## 1. Design Principles

### 1.1 Core Principles
- **Accessibility First**: WCAG 2.1 AA compliance minimum
- **Role-Based Navigation**: Users only see what they're authorized to access
- **Responsive Design**: Desktop (primary), tablet (secondary), mobile (future)
- **Performance**: Time to Interactive < 2 seconds
- **Consistency**: Shared component library across all pages

### 1.2 State Management
Every page must implement:
- **Empty State**: No data available
- **Loading State**: Data being fetched
- **Error State**: Request failed
- **Success State**: Normal operation with data

## 7. Accessibility Requirements

### 7.1 Keyboard Navigation
- Tab order logical
- Skip to main content link
- All actions keyboard accessible
- Focus visible indicators

### 7.2 Screen Reader Support
- Semantic HTML
- ARIA labels where needed
- Status announcements
- Error announcements

### 7.3 Visual Design
- Color contrast ratio: 4.5:1 minimum
- Focus indicators: 3:1 minimum
- Text size: 16px minimum body text
- No information by color alone
