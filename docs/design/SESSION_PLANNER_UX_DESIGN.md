# Session Planner - UX/UI Design Document

**Version:** 1.0
**Date:** February 13, 2026
**Status:** Design Phase
**Target:** Phase 1 Individualized Tutoring Platform

---

## Executive Summary

The Session Planner is the strategic planning interface for tutors to design effective learning sessions before they occur. It leverages historical data, student profiles, and content recommendations to help tutors create personalized, data-driven session plans that optimize learning outcomes.

---

## Design Principles

1. **Data-Informed Planning** - Surface relevant historical context and patterns
2. **Template-Driven Efficiency** - Reuse successful session structures
3. **Flexible Customization** - Easy adjustments for individual needs
4. **Progressive Disclosure** - Show details when needed, hide complexity when not
5. **Multi-Student Coordination** - Plan for groups with different needs
6. **Seamless Integration** - Connect planning to execution (Dashboard)

---

## User Personas

### Primary User: Tutor/Teacher (TEACHER role)
- Plans 5-20 sessions per week
- Manages students at different levels working on different content
- Wants to reuse successful session patterns
- Needs to see student progress history before planning
- Balances individual needs with group dynamics
- Tracks content completion and assignment follow-up

### Secondary User: Center Admin/Supervisor
- Reviews session plans for quality assurance
- Creates center-wide session templates
- Monitors tutor planning patterns
- Ensures curriculum alignment

## Edge Cases & Error Handling

### Conflicting Sessions
**Issue:** Tutor tries to schedule session that overlaps with existing one

**Handling:**
- Red warning banner: "Conflict detected: You have a session with John at 2:00 PM"
- Show conflicting session details
- Options: Choose different time, Cancel conflicting session, Proceed anyway (with reason)

### Student Unavailable
**Issue:** Student has another session or marked as unavailable

**Handling:**
- Yellow warning badge on student card: "Unavailable: Already scheduled at this time"
- Disable checkbox for selection
- Tooltip: "Student has Math session with Tutor B at 2:00 PM"

### Missing Prerequisites
**Issue:** Exercise requires content that student hasn't completed

**Handling:**
- Orange warning on exercise recommendation
- "âš ï¸ Prerequisite gap: Student hasn't completed Unit 2, Lesson 4"
- Options: Add prerequisite first, Choose different exercise, Override (with note)

### Network Issues
**Issue:** Save fails due to connectivity

**Handling:**
- Retry automatically (3 attempts)
- If still failing: Save to local storage
- Banner: "Offline mode: Changes will sync when connection restored"
- Periodic retry in background

### Template Conflicts
**Issue:** Template uses deprecated exercises or content

**Handling:**
- Warning on template selection: "This template references 2 unavailable exercises"
- Auto-suggest replacements from same unit/difficulty
- Allow tutor to fix and update template

### Concurrent Edits
**Issue:** Two tutors (or admin) edit same session simultaneously

**Handling:**
- Lock session when opened for editing (5-minute timeout)
- If locked: "Session being edited by Tutor A. [View Only] [Request Control]"
- On save conflict: Show diff, merge options, force overwrite (with warning)
